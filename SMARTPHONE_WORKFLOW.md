# スマホから改修依頼する運用

このリポジトリは GitHub Codespaces で開けるようにしています。

## 目標

PCを起動していなくても、スマホから改修依頼を出し、変更確認までは進められる状態にします。

## 基本の流れ

1. スマホで GitHub の `rice-os` リポジトリを開く
2. Codespaces を起動する
3. Codex / ChatGPT に改修内容を伝える
4. 変更、確認、コミット手前まで進める
5. 問題なければ「pushして」と伝える

## プレビュー

Codespaces 内で以下を実行すると、ブラウザで確認できます。

```bash
python3 -m http.server 8001
```

表示するファイルは通常 `index.html` です。

## 大事な注意

アプリの入力データはスマホのブラウザ内に保存されます。
改修しても通常は消えませんが、大きな改修前はアプリの「データ管理」から JSON 保存しておくと安全です。

## コミットとプッシュ

作業中は原則として push しません。
最後に確認してから、ユーザーが「pushして」と言った時だけ push します。
